## 一、总提示词

你是一名资深移动端产品 UI 设计师。请为「父母指南」微信小程序生成一组高完成度设计稿。

产品定位：父母通过场景化题目了解自己的育儿风格。它不是考试，不是诊断，不评判父母，而是让家长在真实育儿场景里看见自己的第一反应，并得到温和、具体、可执行的建议。

目标用户：0-18 岁孩子家长，优先考虑 0-6 岁新手父母。用户通常有焦虑、内疚、疲惫、想变好但不想被说教的心理状态。

设计关键词：温暖、安全感、不评判、陪伴感、真实生活、手绘温度、微信小程序、移动端优先。

禁止方向：不要科技感、不要 AI 感、不要冷色渐变、不要未来感、不要仪表盘风格、不要机器人、不要芯片、不要几何抽象、不要模板化营销首屏、不要夸张装饰、不要空洞大字报。

整体视觉系统：
- 画布尺寸：375x812px，iPhone X 比例，PNG。
- 背景色：#F5F0EB。
- 卡片色：#FFFFFF。
- 主文字：#1A1A1A。
- 次文字：#666666。
- 辅助文字：#999999。
- 分隔线：#E5E5E5。
- 页面左右边距：20px。
- 卡片圆角：16-20px。
- 按钮圆角：24-28px。
- 卡片阴影：很轻，纸张叠放感，不要厚重阴影。
- 字体：中文圆润、人文、亲和，标题清楚，正文可读。
- 最小字号：12px。
- 插画：手绘水彩、彩铅或绘本风，边缘略不规整，有真实表情，不要矢量扁平图。

页面强调色：
- 首页：暖橙 #FF8C42。
- 信息收集页：暖杏 #FFB366。
- 答题页：暖米 #F5E6D3。
- 结果页：暖金 #D4A574。
- 心路历程页：暖绿 #8FBC8F。
- 海报分享页：暖橙 #FF8C42。
- 课程推荐页：暖蓝 #6B8E8B。
- 历史记录页：暖紫 #9B7EBD。

统一交付要求：
- 每张图必须是完整手机页面，不是局部组件。
- 所有文字用中文。
- 页面里不要出现“AI”“智能算法”“精准诊断”等表达。
- 不要使用 emoji 字符；需要图标时画成手绘小图标。
- 文字必须清晰可读，不能糊成装饰。
- UI 元素不能重叠，按钮文字不能溢出。
- 设计要能被前端照着实现，不要过度艺术化。

## 四、页面 03：答题页 page_03_quiz.png

生成答题页设计稿。

页面目标：让用户进入真实育儿场景，选择第一反应。不要让它像考试。

页面内容：

## 五、页面 04：结果页 page_04_result.png

生成结果页设计稿，展示“共情引导者”画像。

页面目标：让用户获得“被理解”的瞬间，并知道下一步怎么做。

页面内容：
- 顶部手绘插画：家长蹲下来和孩子平视交流，温暖但不溺爱。
- 标签：测试结果。
- 画像名：共情引导者。
- 金句：先懂心，再引路。
- 四维度分布：
  - A 情绪回应 8次。

  - C 探索追问 7次。
  - D 行动果断 3次。
- 分数条或简化雷达图。

- 明天就能做的一件事：孩子情绪上来时，先说“我看见你很难过”，再问“你想自己说，还是我陪你说？”

- 次按钮：查看行动清单。

设计要求：
- 暖金 #D4A574 为主强调色。
- 结果页不要只像证书，要像一份温和但具体的反馈。
- “容易卡住”不能像批评，要像提醒。
- 页面可设计为长页，但 375x812 内要呈现主要内容。

输出文件名：`page_04_result.png`。
