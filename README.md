# My AI Workbench

这是一个给多智能体协作使用的工作台。群聊只负责调度，仓库负责记录，任务票据负责闭环，预授权规则负责减少反复确认。

## 核心规则

1. 每个任务必须有任务票据。
2. 每个任务只能有一个主负责人。
3. 执行者可以在预授权范围内连续推进，不要每一步都问用户。
4. 遇到高风险动作才请求确认。
5. 每次完成任务后必须回填结果、产出、问题和学到的偏好。

## 目录说明

- `rules/`: 长期规则，包括角色、授权、工作流、禁止事项。
- `tasks/active/`: 正在做的任务。
- `tasks/done/`: 已完成任务。
- `briefs/`: 需求、设计稿分析、项目背景。
- `prompts/`: 图片生成、代码生成、内容生成提示词。
- `outputs/`: 最终产物链接、截图说明、发布记录。
- `logs/`: 日报、周报、执行记录。
- `decisions/`: 重要决策和原因。
- `memory/`: 用户偏好、风格、代码标准、社媒风格。

## 每日用法

早上发给 Kimi：

```text
Kimi，读取 my-ai-workbench。按 rules/workflow_rules.md 调度今日任务。除 rules/approval_policy.md 中必须确认的事项外，不要打扰我。每 2 小时只汇报阻塞、待确认和已完成产出。
```

晚上发给 PC-Claw：

```text
PC-Claw，读取 tasks/active 中分配给 PC-Claw 的任务。按任务票据执行，完成后提交结果并更新日志。
```
